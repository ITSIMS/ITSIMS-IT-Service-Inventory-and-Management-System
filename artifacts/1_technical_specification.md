## 1. Цель документа

Данный документ определяет требования заказчика к Автоматизированной информационной системе учёта и управления IT-сервисами (ITSIMS). 

Документ предназначен для заказчика (АО «БАРС Груп»), команды разработки, аналитиков и других заинтересованных сторон проекта.

Из документа читатель узнает о назначении системы, её функциональных возможностях, бизнес-требованиях к управлению каталогом IT-сервисов, документацией, правами доступа и обслуживанием сервисов.

**Принцип:** Техническое задание отвечает на вопрос "**ЧТО** должна делать система с точки зрения бизнеса", не вдаваясь в технические детали реализации.



## 2. Термины, определения и сокращения

### 2.1. Базовые термины

- **Заказчик** — АО «БАРС Груп» в лице представителя Суставова Сергея Александровича

- **Система** — Автоматизированная информационная система учёта и управления IT-сервисами (ITSIMS), разрабатываемая в рамках данного проекта

- **Пользователь** — сотрудник организации-заказчика, использующий систему в рамках своих должностных обязанностей и предоставленных прав доступа

- **Администратор системы** — пользователь с полным контролем над системой, включая управление всеми сервисами, категориями, правами доступа и настройками системы. Роль технического директора автоматически обладает правами администратора системы

### 2.2. Специфичные термины

- **IT-сервис** — единица учёта в системе, представляющая собой программное обеспечение с определённым набором характеристик (название, описание, версия, категория, статус, привязка к аппаратам) и ассоциированной документацией

- **Администратор сервиса** — пользователь, обладающий полными правами на управление конкретным IT-сервисом: редактирование характеристик, управление документацией, распределение прав доступа к данному сервису

- **Специалист по обслуживанию** — пользователь, обладающий правами на просмотр документации по обслуживанию сервиса и регистрацию фактов обслуживания

- **Обычный сотрудник** — пользователь с правами на просмотр доступных ему IT-сервисов и документации для пользователей (инструкции по установке, настройке, использованию)

- **Категория сервиса** — классификационный признак IT-сервиса, определяемый администратором системы на основе справочника категорий

- **Связь сервисов** — однонаправленная зависимость между IT-сервисами, указывающая, что один сервис использует другой для своей работы

- **Граф зависимостей** — визуальное представление IT-сервисов и связей между ними в виде ориентированного графа

- **Файловый каталог сервиса** — структурированное хранилище документации, ассоциированное с конкретным IT-сервисом

- **Документ для пользователей** — файл в каталоге сервиса, содержащий информацию для обычных сотрудников (руководства пользователя, инструкции по установке и настройке)

- **Документ для обслуживания** — файл в каталоге сервиса, содержащий информацию для специалистов по обслуживанию и администраторов (техническая документация, регламенты обслуживания)

- **Запись обслуживания** — фиксация факта проведения работ по обслуживанию IT-сервиса с указанием даты, описания работ и исполнителей

- **Статус сервиса** — текущее состояние IT-сервиса в жизненном цикле (активный, архивный, на обслуживании, в разработке, на тестировании)

- **Аппарат** — единица аппаратного обеспечения организации (физический или виртуальный сервер, хост), на котором установлен, хранится или выполняется IT-сервис

## 4. Назначение и цели создания системы

### 4.1. Назначение системы

Система предназначена для централизованного учёта и управления IT-сервисами организации АО «БАРС Груп». Система обеспечивает структурированное хранение информации о программном обеспечении, используемом в организации, связях между сервисами, документации и правах доступа к ней.

Система используется сотрудниками IT-отдела для управления каталогом сервисов и обслуживания, а также обычными сотрудниками организации для самостоятельного получения доступа к документации по установке и настройке необходимого программного обеспечения.

### 4.2. Цели создания системы

Основные цели создания системы:

1. Централизация учёта IT-сервисов организации в единой информационной системе
2. Установление и визуализация связей между IT-сервисами для понимания зависимостей
3. Снижение трудозатрат на поддержку и обслуживание внедрённых IT-сервисов
4. Автоматизация процессов внесения новых сервисов и архивирования устаревших
5. Обеспечение разграничения доступа к документации сервисов в зависимости от роли пользователя
6. Предоставление сотрудникам организации возможности самостоятельной установки и настройки IT-сервисов на основе документации

- <a id="ts-f038"></a>**TS-F038:** Система должна обеспечивать возможность регистрации факта обслуживания IT-сервиса специалистом по обслуживанию [↓](./2_TS_to_SR.md#ts-f038-down)

- <a id="ts-f039"></a>**TS-F039:** Система должна фиксировать в записи обслуживания следующую информацию: дата проведения работ, описание выполненных работ, список исполнителей [↓](./2_TS_to_SR.md#ts-f039-down)

- <a id="ts-f040"></a>**TS-F040:** Система должна обеспечивать возможность просмотра истории обслуживания IT-сервиса пользователями с правами на обслуживание данного сервиса [↓](./2_TS_to_SR.md#ts-f040-down)

- <a id="ts-f041"></a>**TS-F041:** Система должна отображать дату последнего обслуживания IT-сервиса специалистам по обслуживанию [↓](./2_TS_to_SR.md#ts-f041-down)

- <a id="ts-nf006"></a>**TS-NF006:** Система должна обеспечивать разграничение прав доступа пользователей на основе их ролей [↓](./2_TS_to_SR.md#ts-nf006-down)

- <a id="ts-nf007"></a>**TS-NF007:** Система должна обеспечивать защиту от несанкционированного доступа к документации сервисов [↓](./2_TS_to_SR.md#ts-nf007-down)

- <a id="ts-nf008"></a>**TS-NF008:** Система должна обеспечивать аутентификацию пользователей при входе в систему [↓](./2_TS_to_SR.md#ts-nf008-down)

- <a id="ts-nf009"></a>**TS-NF009:** Система должна обеспечивать идентификацию пользователя и невозможность представиться другим сотрудником [↓](./2_TS_to_SR.md#ts-nf009-down)

- <a id="ts-nf010"></a>**TS-NF010:** Система должна обеспечивать отображение пользователю только тех IT-сервисов, к которым у него действительно есть права доступа согласно настроенным разрешениям [↓](./2_TS_to_SR.md#ts-nf010-down)

- <a id="ts-nf011"></a>**TS-NF011:** Система должна обеспечивать защиту от несанкционированного получения доступа к IT-сервисам и документации путём изменения параметров запроса или манипуляции сеансом [↓](./2_TS_to_SR.md#ts-nf011-down)

- <a id="ts-nf013"></a>**TS-NF013:** Система должна обеспечивать удобную навигацию по каталогу сервисов и документации [↓](./2_TS_to_SR.md#ts-nf013-down)
