## Назначение директории

Директория `artifacts/` содержит **конечные артефакты проекта** — финальные версии документов, готовых к сдаче.

**Что здесь хранится:**
- Техническое задание (ТЗ)
- Требования к системе
- Требования к программному обеспечению
- Таблицы трассировки требований между артефактами

**Что здесь НЕ хранится:**
- Черновики и промежуточные версии
- Документы для внутреннего использования команды
- Временные файлы



## Структура директории

```
artifacts/
├── README.md                                 # Данный документ
├── 1_technical_specification.md              # Техническое задание (ТЗ)
├── 2_TS_to_SR.md                             # Трассировка: ТЗ ↔ Требования к системе
├── 3_system_requirements.md                  # Требования к системе
├── 4_SR_to_SWR.md                            # Трассировка: Требования к системе ↔ Требования к ПО
├── 5_software_requirements.md                # Требования к программному обеспечению
└── _formatting/                              # Правила и шаблоны для артефактов
    ├── 1_technical_specification_rules.md    # Правила оформления ТЗ
    ├── 2_system_requirements_rules.md        # Правила оформления SR
    ├── 3_software_requirements_rules.md      # Правила оформления SWR
    └── general_artifact_rules.md             # Общие правила оформления артефактов
```

**Логика нумерации:**
- Артефакты и файлы трассировки нумеруются последовательно (1, 2, 3, 4, 5)
- Файлы трассировки размещаются **между** артефактами, которые они связывают
- Это отражает естественный порядок чтения: ТЗ → связь → Требования к системе → связь → Требования к ПО



## Правила работы с артефактами

### Нумерация файлов

Файлы нумеруются префиксами `1_`, `2_`, `3_` для определения порядка:

```
✅ 1_technical_specification.md # Первый документ - ТЗ
✅ 2_system_requirements.md     # Второй документ - требования
✅ 3_architecture.md            # Третий документ (будущий)
```

Артефакты создаются последовательно, поэтому нумерация отражает эту последовательность.

### Версионирование

Версии документов отслеживаются через **Git**, а не через номера в именах файлов:

```
✅ artifacts/1_technical_specification.md
   (история изменений в Git)

❌ artifacts/1_technical_specification_v1.md
❌ artifacts/1_technical_specification_v2.md
❌ artifacts/1_technical_specification_final.md
```

### Служебные папки

Папки с шаблонами и правилами начинаются с подчеркивания:

```
✅ _formatting/  # Служебная папка
```

Это гарантирует, что такие папки отображаются в начале списка.
