## Назначение папки

Папка `project_rules/` содержит **общие правила при создании и оформлении документов** проекта ITSIMS. Эти правила применимы ко всем типам документов в репозитории: артефактам, планам, отчётам, техническим документам и прочим типам, которые могут появиться в процессе работы над проектом.

**Цель:** Обеспечить единообразие, читаемость и профессиональное качество всей документации проекта.



## Структура папки

```
project_rules/
├── README.md                      # Описание правил и структуры папки
├── format_selection.md            # Правила выбора формата документа (Markdown, LaTeX)

├── formatting_principles.md       # Общие принципы оформления и форматирования документов
└── markdown_guide.md              # Применение общих принципов оформления на примере Markdown
```



## Описание документов

### 1. [Выбор формата документа](./format_selection.md)

**Когда использовать:** Перед началом работы над любым новым документом.

**Содержание:**
- Markdown как основной формат проекта
- Когда использовать Markdown
- Когда использовать LaTeX
- Правила переключения между форматами

**Ключевой принцип:** Используй самый простой формат, который решает задачу.



### 2. [Структура документа](./document_structure.md)

**Когда использовать:** При создании нового документа или проверке существующего.

**Содержание:**
- Обязательные элементы документа:
  - Заголовок документа
  - Таблица версионирования
  - Оглавление (когда необходимо)
  - Основное содержание
- Рекомендуемая последовательность разделов
- Правила структурирования больших документов
- Особенности структуры для разных типов документов

**Ключевой принцип:** Структура документа должна быть предсказуемой и помогать читателю быстро находить нужную информацию.



### 3. [Общие принципы оформления документов](./formatting_principles.md)

**Когда использовать:** При написании и редактировании любого документа.

**Содержание:**
- Структура документа и иерархия заголовков
- Правила выделения текста (жирный, курсив, код)
- Оформление списков, таблиц, ссылок, изображений
- Форматирование кода и математических формул
- Правила именования файлов и организации документов

**Ключевой принцип:** Форматирование должно помогать читателю, а не отвлекать его.



### 4. [Руководство по Markdown для проекта ITSIMS](./markdown_guide.md)

**Когда использовать:** При работе с документами в формате Markdown (основной формат проекта).

**Содержание:**
- Синтаксис Markdown с примерами
- Выделение текста (жирный, курсив, код)
- Создание списков и таблиц
- Работа со ссылками и изображениями
- Математические формулы в Markdown
- Блоки кода с подсветкой синтаксиса
- Специальные блоки (примечания, предупреждения)

**Ключевой принцип:** Markdown должен оставаться читаемым в исходном виде.



## Рекомендации по использованию для новых участников проекта

3. **Используй как справочник:** [Общие принципы оформления](./formatting_principles.md) — универсальные правила
