## 1. Цель документа

Данное руководство определяет единые стандарты форматирования документации проекта ITSIMS. Цель — обеспечить читаемость, единообразие и простоту поддержки всех документов проекта.

**Принципы:**
- **Простота** — минимум формализма, максимум пользы
- **Единообразие** — один стиль для всех типов документов
- **Git-friendly** — форматы, удобные для версионирования
- **Автоматизация** — возможность генерации PDF при необходимости

## 2. Выбор формата

### 2.1 Markdown — основной формат

**Все документы проекта создаются в формате Markdown (`.md`).**

**Почему Markdown:**
- ✅ Читаем в исходном виде (открывается в любом редакторе)
- ✅ Понятные изменения в Git (текстовый diff)
- ✅ Простота редактирования (не нужны специальные программы)
- ✅ Легко конвертировать в PDF через Pandoc
- ✅ Поддержка таблиц, списков, кода, ссылок

**Когда использовать:**
- Все артефакты (ТЗ, требования, планы)
- Документация проекта
- Отчёты о валидации и инспекциях
- Задачи и еженедельные отчёты
- README и инструкции

### 2.2 LaTeX — для исключительных случаев

**LaTeX используется только когда Markdown не справляется.**

**Когда можно использовать LaTeX:**
- Сложные математические формулы (редко нужно в нашем проекте)
- Специфичное оформление с точным контролем вёрстки
- Требования заказчика к формату документа

**⚠️ Важно:** Перед использованием LaTeX нужно:
1. Убедиться, что Markdown не справляется с задачей
2. Обсудить необходимость с командой
3. Указать причину выбора LaTeX в истории версий документа

**Пример неудачного использования LaTeX:**
- ❌ "Хочу красивые таблицы" → используй Markdown
- ❌ "Нужна нумерация страниц" → используй Pandoc для конвертации Markdown в PDF
- ❌ "Заказчику нужен PDF" → конвертируй Markdown в PDF

**Пример удачного использования LaTeX:**
- ✅ Документ содержит сложные формулы теории вероятностей
- ✅ Требуется специфичная вёрстка по ГОСТ, которую не поддерживает Pandoc
- ✅ Заказчик требует LaTeX как исходный формат
