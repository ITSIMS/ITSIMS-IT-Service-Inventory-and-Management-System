## 1. Цель документа

Данный документ определяет правила выбора формата для документации проекта ITSIMS. Цель — использовать наиболее подходящий формат для каждой задачи, избегая избыточной сложности.

**Принцип:** Используй самый простой формат, который решает задачу.



## 2. Markdown — основной формат проекта

**Все документы проекта по умолчанию создаются в формате Markdown (`.md`).**

### 2.1 Преимущества Markdown

- ✅ **Читаемость** — понятен в исходном виде, открывается в любом редакторе
- ✅ **Git-friendly** — текстовый формат обеспечивает понятные diff'ы
- ✅ **Простота** — не требует специальных программ для редактирования
- ✅ **Конвертация** — легко преобразовать в другой формат через Pandoc
- ✅ **Универсальность** — поддержка таблиц, списков, кода, формул, ссылок

### 2.2 Когда использовать Markdown

- По умолчанию для **любого** текстового документа проекта

**Правило:** Если сомневаешься — используй Markdown.



## 3. LaTeX — для исключительных случаев

**LaTeX используется только когда Markdown и Pandoc не справляются с задачей.**

### 3.1 Когда допустимо использовать LaTeX

- **Сложные математические формулы** (редко актуально для нашего проекта)
- **Специфичная вёрстка по ГОСТ**, которую не поддерживает Pandoc
- **Требования заказчика** к LaTeX как исходному формату документа

### 3.2 Процесс принятия решения

Перед использованием LaTeX необходимо:

1. **Попробовать Markdown** — убедиться, что он не справляется
2. **Попробовать Pandoc** — проверить возможности конвертации с шаблонами
3. **Обсудить с командой** — согласовать необходимость LaTeX
4. **Документировать причину** — указать в версионной таблице, почему выбран LaTeX
