# Структурировать папку validations и создать регулирующий документ

**Создана:** 2025-11-23
**Ответственный:** Зацепилин
**Нужно сделать до:** 2025-11-28

## Описание

Привести папку `validations/` в соответствие с общими правилами проекта. В настоящий момент в папке:
- Все документы в формате PDF (нарушение принципа "исходники в markdown")
- Отсутствует регулирующий документ с правилами проведения валидаций и инспекций
- Неясна общая структура и процесс проведения валидаций

**Ожидаемый результат:**
- Создан регулирующий документ `validations/README.md` с описанием процесса валидаций/инспекций
- Все существующие PDF-отчеты преобразованы в markdown (или созданы исходные markdown-версии)
- Шаблон отчета о формальной инспекции преобразован в markdown
- Документация содержит версионные таблицы согласно общим правилам проекта

## Чеклист

### 1. Создать регулирующий документ `validations/README.md`

- [ ] Описать цель и назначение папки `validations/`
- [ ] Описать типы валидаций/инспекций (формальные инспекции, ревью кода, проверка документов и т.д.)
- [ ] Определить процесс проведения инспекций:
  - [ ] Кто инициирует
  - [ ] Кто проводит
  - [ ] Как оформляются результаты
  - [ ] Куда сохраняются отчеты
- [ ] Описать структуру папки и правила именования файлов
- [ ] Указать ответственных и роли в процессе валидации
- [ ] Добавить ссылки на шаблоны и примеры
- [ ] Добавить версионную таблицу в начало документа

### 2. Преобразовать PDF-файлы в markdown

- [ ] Изучить существующие PDF:
  - `validations/_formatting/formal_inspection_report_template.pdf`
  - `validations/reports/2025-11-03_spec_technical_formal_inspection_report_v1.0.pdf`
- [ ] Создать markdown-версию шаблона:
  - [ ] `validations/_formatting/formal_inspection_report_template.md`
  - [ ] Добавить версионную таблицу
  - [ ] Структурировать шаблон согласно общим правилам оформления
- [ ] Создать markdown-версию отчета:
  - [ ] `validations/reports/2025-11-03_spec_technical_formal_inspection_report_v1.0.md`
  - [ ] Добавить версионную таблицу
  - [ ] Перенести содержимое из PDF

### 3. Проверить соответствие общим правилам

- [ ] Убедиться, что все новые markdown-файлы содержат версионные таблицы
- [ ] Проверить соответствие форматированию из `project_rules/unified_document_format.md`
- [ ] Убедиться, что используется русский язык
- [ ] Проверить корректность ссылок между документами

### 4. Решить вопрос с PDF-файлами

- [ ] Обсудить с командой: нужны ли PDF-версии отчетов о валидации?
- [ ] Если нужны - добавить их в `.gitignore` и генерировать через `tools/printing/`
- [ ] Если не нужны - удалить существующие PDF после создания markdown-версий

### 5. Финальная проверка

- [ ] Проверить, что структура папки логична и понятна
- [ ] Убедиться, что документация достаточна для новых участников команды
- [ ] Проверить все внутренние ссылки
- [ ] Провести самопроверку: можешь ли ты по своему `README.md` провести валидацию?
