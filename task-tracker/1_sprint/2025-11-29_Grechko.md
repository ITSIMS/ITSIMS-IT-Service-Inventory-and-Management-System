- [ ] Создать `general_rules/naming_conventions.md` - правила именования файлов и папок:
  - Префиксы для сортировки (`_`, `1_`, `2_`, ...)
  - Формат имён файлов в разных папках
  - Правила именования задач
  - Запрещённые символы и практики
- [ ] Создать `general_rules/file_organization.md` - принципы организации файлов:
  - Разделение на инструменты/инструкции/шаблоны/документы
  - Где какие типы файлов размещать
  - Правила создания новых папок
- [ ] Создать `general_rules/git_workflow.md` - правила работы с Git:
  - Использование `git mv` для перемещения файлов
  - Правила коммитов (ссылка на `.cursor/rules/2_git/commit_format.mdc`)
  - Работа с ветками (если применимо)
  - Интеграция с task-tracker
- [ ] Удалить устаревшие файлы из `general_rules/`:
  - `decision_document_format.md`
  - `unified_document_format.md`
- [ ] Создать `general_rules/README.md` - навигацию по правилам

### 2. Финализация
- [ ] Проверить корректность всех внутренних ссылок
- [ ] Убедиться, что документация согласована
- [ ] Проверить оформление по правилам Markdown

## Примечания

### Структура general_rules/ (целевая)
```
general_rules/
├── README.md                      # Навигация по правилам
├── naming_conventions.md          # Правила именования
├── file_organization.md           # Организация файлов
└── git_workflow.md                # Работа с Git
```

### Требования к документам
- Должны быть краткими и информативными
- Содержать минимум необходимой информации для старта
- Ссылки на детальные документы, а не полное их содержание
- Визуально структурированы (заголовки, списки, таблицы)
