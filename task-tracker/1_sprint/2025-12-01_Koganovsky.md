# Создание правил оформления и шаблона технического задания

**Создана:** 2025-12-01
**Ответственный:** Когановский
**Нужно сделать до:** 2025-12-02

## Описание

Переход от общих правил к частным: создать конкретные правила оформления технического задания и работающий шаблон, который можно использовать для создания новых ТЗ.

**Результат:** 
- Документ `artifacts/_formatting/1_technical_rules.md` с правилами
- Шаблон `artifacts/_formatting/1_technical_template.md`

---

## Чеклист

### Этап 1: Правила оформления ТЗ

- [ ] Создать `artifacts/_formatting/1_technical_rules.md`
  - [ ] Назначение и область применения ТЗ
  - [ ] Обязательная структура документа:
    - [ ] Титульная информация
    - [ ] Таблица версионирования
    - [ ] Оглавление
    - [ ] Введение (цели, область применения, термины)
    - [ ] Общее описание системы
    - [ ] Функциональные требования
    - [ ] Нефункциональные требования
    - [ ] Ограничения
    - [ ] Приложения (опционально)
  
  - [ ] Правила идентификации требований ТЗ
    - [ ] Формат ID для функциональных требований: `TS-F-XXX`
    - [ ] Формат ID для нефункциональных требований: `TS-NF-XXX`
    - [ ] Правила нумерации внутри категорий
  
  - [ ] Правила формулировки требований
    - [ ] Использование модальных глаголов (должен, может, следует)
    - [ ] Структура описания требования
    - [ ] Обязательные атрибуты (приоритет, источник)

### Этап 2: Создание шаблона

- [ ] Создать `artifacts/_formatting/1_technical_template.md`
  - [ ] Заполнить структуру с placeholder'ами
  - [ ] Добавить комментарии-подсказки для каждого раздела
  - [ ] Создать примеры требований с правильными ID
  - [ ] Добавить пример таблицы версионирования
  - [ ] Подготовить заготовку для оглавления

- [ ] Убедиться в применимости шаблона
  - [ ] Шаблон должен легко копироваться и заполняться
  - [ ] Все обязательные элементы присутствуют
  - [ ] Формат совместим с существующим `artifacts/1_technical_specification.md`

### Этап 3: Интеграция с общими правилами

- [ ] Связать правила ТЗ с общими правилами
  - [ ] Ссылка на `general_artifact_rules.md` (система трассировки)
  - [ ] Ссылка на общие правила оформления документов из `project_rules/`
  - [ ] Пометить специфичные для ТЗ моменты

- [ ] Создать краткую инструкцию по использованию
  - [ ] Как использовать шаблон
  - [ ] Как правильно присваивать ID требованиям
  - [ ] Как поддерживать таблицу версионирования

---

## Критерии приемки

1. Файл `artifacts/_formatting/1_technical_rules.md` создан и содержит все необходимые правила
2. Шаблон `artifacts/_formatting/1_technical_template.md` готов к использованию
3. Правила ID для ТЗ четко определены (TS-F-XXX, TS-NF-XXX)
4. Шаблон содержит примеры и комментарии-подсказки
5. Правила согласованы с общими правилами из предыдущих задач

## Примечания

**Базис:** Эта задача опирается на:
- Общие правила оформления документов (задача 2025-11-29)
- Общие правила артефактов и трассировки (задача 2025-11-30)

**ТЗ — верхний уровень:** Техническое задание содержит высокоуровневые требования заказчика. Из ТЗ будут детализироваться требования к системе, а затем требования к ПО.

**Шаблон должен работать:** Основная цель — создать практичный инструмент. Шаблон должен быть готов к немедленному использованию.
