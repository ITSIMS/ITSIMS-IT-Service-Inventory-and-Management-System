# Привести ссылки в таблицах трассировки системных требований

**Создана:** 2025-12-09
**Ответственный:** Когановский
**Нужно сделать до:** 2025-12-14

## Описание

Необходимо привести в порядок таблицы трассировки между требованиями разных уровней. Задача включает обновление существующих таблиц и обеспечение корректных двусторонних ссылок между техническим заданием, системными требованиями и программными требованиями.

**Контекст:**
- Вальковец создал системные требования с идентификаторами SR-XXX
- Существуют таблицы трассировки, которые необходимо актуализировать
- Таблицы должны обеспечивать прослеживаемость требований на всех уровнях

**Зависимости:**
- Дождаться завершения задачи Вальковца (создание системных требований)

**Ожидаемый результат:**
- Актуализированная таблица трассировки `artifacts/traceability/TS_to_SR.md`
- Подготовленная таблица трассировки `artifacts/traceability/SR_to_SWR.md`
- Все ссылки корректны и взаимосвязаны
- Таблицы готовы к формальной инспекции

**Связанные файлы:**
- `artifacts/1_technical.md` — техническое задание
- `artifacts/2_system_requirements.md` — системные требования
- `artifacts/traceability/TS_to_SR.md` — таблица трассировки ТЗ→СТ
- `artifacts/traceability/SR_to_SWR.md` — таблица трассировки СТ→ПТ

## Чеклист

### Этап 1: Подготовка

- [ ] Дождаться завершения задачи Вальковца (создание системных требований)
- [ ] Изучить созданные системные требования и их идентификаторы
- [ ] Проанализировать существующие таблицы трассировки
- [ ] Определить формат и структуру таблиц

### Этап 2: Обновление таблицы TS_to_SR

- [ ] Открыть файл `artifacts/traceability/TS_to_SR.md`
- [ ] Обновить титульную информацию (версия, дата)
- [ ] Для каждого требования из ТЗ:
  - [ ] Проверить идентификатор (TS-XXX)
  - [ ] Указать все связанные системные требования (SR-XXX)
  - [ ] Убедиться, что ссылки корректны
  - [ ] Добавить краткое описание связи (при необходимости)
- [ ] Проверить полноту покрытия:
  - [ ] Все требования из ТЗ присутствуют в таблице
  - [ ] Нет "потерянных" требований
- [ ] Обновить таблицу версионирования

### Этап 3: Подготовка таблицы SR_to_SWR

- [ ] Создать/обновить файл `artifacts/traceability/SR_to_SWR.md`
- [ ] Заполнить титульную информацию
- [ ] Для каждого системного требования:
  - [ ] Указать идентификатор (SR-XXX)
  - [ ] Подготовить место для программных требований (SWR-XXX)
  - [ ] Добавить примечания о будущей связи
- [ ] Структурировать таблицу по категориям:
  - [ ] Функциональные требования (SR-F-XXX)
  - [ ] Нефункциональные требования (SR-NF-XXX)
- [ ] Создать таблицу версионирования

### Этап 4: Проверка целостности

- [ ] Проверить прямую трассировку (TS → SR → SWR):
  - [ ] Каждое требование ТЗ ведет к системным требованиям
  - [ ] Каждое системное требование подготовлено для связи с программными
- [ ] Проверить обратную трассировку:
  - [ ] Нет "висящих" системных требований без связи с ТЗ
  - [ ] Все идентификаторы уникальны
- [ ] Проверить формат:
  - [ ] Единообразие оформления таблиц
  - [ ] Корректность ссылок (если используются markdown-ссылки)
  - [ ] Соответствие правилам проекта

### Этап 5: Финализация

- [ ] Проверить оба файла на соответствие правилам оформления
- [ ] Убедиться, что таблицы:
  - [ ] Читаемы
  - [ ] Логически структурированы
  - [ ] Содержат всю необходимую информацию
- [ ] Создать краткую документацию по использованию таблиц (в самих файлах)
- [ ] Подготовить файлы к формальной инспекции

## Примечания

**Важность трассировки:** Таблицы трассировки — ключевой инструмент для:
- Обеспечения полноты покрытия требований
- Анализа влияния изменений
- Валидации реализации
- Формальной инспекции

**Формат идентификаторов:**
- TS-XXX — требования технического задания
- SR-F-XXX — функциональные системные требования
- SR-NF-XXX — нефункциональные системные требования
- SWR-XXX — программные требования (пока пустые)

**Связь с инспекциями:** После создания таблиц Зацепилин проведет их формальную инспекцию отдельно от инспекции документа системных требований.

**Масштабируемость:** Структура таблиц должна позволять легко добавлять новые требования и обновлять связи в будущем.
