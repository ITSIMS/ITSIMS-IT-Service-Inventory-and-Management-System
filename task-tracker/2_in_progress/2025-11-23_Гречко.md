## Примечания

### Структура general_rules/ (целевая)
```
general_rules/
├── README.md                      # Навигация по правилам
├── naming_conventions.md          # Правила именования
├── file_organization.md           # Организация файлов
└── git_workflow.md                # Работа с Git
```

### Требования к README.md
- Должен быть краток и информативен
- Содержать минимум необходимой информации для старта
- Ссылки на детальные документы, а не полное их содержание
- Визуально структурирован (заголовки, списки, таблицы)
