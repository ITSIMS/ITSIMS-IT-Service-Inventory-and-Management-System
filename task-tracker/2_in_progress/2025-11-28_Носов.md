# Реорганизовать план конфигурационного управления

**Создана:** 2025-11-28
**Ответственный:** Носов
**Нужно сделать до:** 2025-12-05

## Описание

План конфигурационного управления (`planning/configuration_management_plan.md`) в настоящее время содержит много встроенных документов, что делает его громоздким и трудным для навигации. Необходимо реорганизовать ПКУ, выделив крупные вложенные документы в отдельные файлы.

**Цель:** Упростить структуру плана КУ, сохранив его читаемость и поддерживаемость.

**Результат:**
- Основной файл `planning/configuration_management_plan.md` содержит ссылки на вынесенные документы
- Отдельные документы для:
  - Правил структуры репозитория (раздел 2.1)
  - Правил наименования файлов (раздел 4.1)
- Обновлен реестр конфигурационных единиц (CIL) с учетом новых файлов
- Основная структура ПКУ сохранена (конфигурационные единицы, роли, процессы)

**Связанные документы:**
- [План конфигурационного управления](../../planning/configuration_management_plan.md)

## Чеклист

- [ ] Создать отдельный документ `planning/repository_structure_rules.md` на основе раздела 2.1 текущего ПКУ
- [ ] Создать отдельный документ `planning/file_naming_rules.md` на основе раздела 4.1 текущего ПКУ
- [ ] Удалить из ПКУ разделы 2.1 и 4.1, заменив их ссылками на новые документы
- [ ] Обновить реестр конфигурационных единиц (раздел 3.2) — добавить новые КЕ, обновить ссылки
- [ ] Обновить таблицу оглавления в ПКУ
- [ ] Обновить лист изменений в ПКУ (добавить версию с текущей датой)
- [ ] Проверить корректность всех ссылок в документах
- [ ] Убедиться, что основная структура ПКУ (роли, КЕ, процессы) осталась неизменной

## Примечания

- При создании новых документов используй стандартную структуру с листом изменений
- Сохрани форматирование и нумерацию существующих разделов
- Убедись, что все ссылки в CIL указывают на правильные файлы или разделы
