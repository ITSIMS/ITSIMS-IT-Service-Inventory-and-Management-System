## Описание

Создать комплексную документацию в папке `project_rules/` для участников команды. Необходимо осмыслить и переработать правила из `.cursor/rules/` в понятную форму для реальных людей.

Создать три ключевых документа:
1. **Работа с Git** - как команда работает с репозиторием
2. **Структура репозитория** - где что размещается
3. **Правила именования** - соглашения об именовании файлов и папок

Задача является продолжением актуализации документации проекта.

**Результат:** Структурированная документация, позволяющая новым участникам быстро разобраться в работе с репозиторием и организации файлов в проекте.

## Примечания

### Структура project_rules/ (целевая)
```
project_rules/
├── README.md                      # Навигация по правилам
├── git_workflow.md                # Работа с Git
├── repository_structure.md        # Структура репозитория
├── naming_conventions.md          # Правила именования
├── decision_document_format.md    # (существующий, не трогать)
└── unified_document_format.md     # (существующий, не трогать)
```

### Источники информации
- `.cursor/rules/` - правила для ИИ агента (использовать как источник информации)
- `task-tracker/README.md` - пример хорошего регулирующего документа
- `README.md` проекта - описание структуры верхнего уровня
- Текущая структура репозитория

### Требования к документам
- Должны быть написаны для реальных участников команды, понятным языком
- Содержать примеры использования и практические сценарии
- НЕ просто копировать или ссылаться на `.cursor/rules/`
- Переработать правила из `.cursor/rules/` в доступную форму
- Визуально структурированы (заголовки, списки, таблицы)
- Написаны на русском языке
- Фокус на практическое применение, а не на технические детали
