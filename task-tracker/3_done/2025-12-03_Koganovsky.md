# Проверка и приведение технического задания к правилам оформления

**Создана:** 2025-12-03
**Ответственный:** Когановский
**Нужно сделать до:** 2025-12-04

## Описание

После того как Вальковец наполнит ТЗ содержанием, необходимо проверить документ на соответствие правилам оформления и привести его к единым стандартам проекта.

**Объект работы:** `artifacts/1_technical_specification.md`

**Базовые правила:**
- `artifacts/_formatting/1_technical_rules.md` (правила для ТЗ)
- `artifacts/_formatting/general_artifact_rules.md` (общие правила артефактов)
- `project_rules/` (общие правила оформления документов)

---

## Чеклист

### Этап 1: Проверка структуры документа

- [ ] Проверить наличие обязательных элементов
  - [ ] Таблица версионирования в начале документа
  - [x] Оглавление (если документ большой)
  - [x] Введение с целями и областью применения
  - [x] Функциональные требования
  - [x] Нефункциональные требования

- [x] Проверить порядок разделов
  - [x] Разделы следуют в логичном порядке
  - [ ] Нумерация разделов корректна
  - [x] Структура соответствует правилам

### Этап 2: Проверка требований

- [x] Проверить идентификаторы требований
  - [x] Функциональные требования имеют ID вида `TS-F-XXX`
  - [x] Нефункциональные требования имеют ID вида `TS-NF-XXX`
  - [x] ID уникальны и последовательны
  - [ ] Каждому требованию присвоен якорь для трассировки

- [x] Проверить формулировки требований
  - [x] Используются правильные модальные глаголы (должен, может, следует)
  - [x] Требования четко сформулированы
  - [x] Требования проверяемы

- [x] Проверить атрибуты требований
  - [x] Указаны приоритеты (если предусмотрено правилами)
  - [x] Указаны источники требований (если предусмотрено правилами)

### Этап 3: Форматирование

- [x] Проверить форматирование Markdown
  - [x] Заголовки используют корректные уровни (H1-H6)
  - [x] Таблицы оформлены единообразно
  - [x] Списки оформлены корректно
  - [ ] Ссылки и якоря работают

- [x] Проверить таблицу версионирования
  - [x] Формат таблицы соответствует правилам
  - [x] Все поля заполнены корректно
  - [x] История изменений актуальна

### Этап 4: Финализация

- [ ] Исправить найденные несоответствия
  - [ ] Добавить недостающие элементы
  - [ ] Присвоить ID требованиям без них
  - [ ] Исправить неправильное форматирование
  - [ ] Привести таблицы к единому стилю

- [ ] Обновить метаинформацию
  - [ ] Актуализировать оглавление (если есть)
  - [ ] Обновить таблицу версионирования
  - [ ] Добавить запись о проверке форматирования

---

## Критерии приемки

1. Документ `artifacts/1_technical_specification.md` полностью соответствует правилам оформления
2. Все требования имеют корректные ID формата TS-F-XXX и TS-NF-XXX
3. Структура документа соответствует правилам для ТЗ
4. Таблица версионирования актуальна
5. Форматирование Markdown единообразно и корректно
6. Все обязательные элементы присутствуют

---

## Примечания

**Зависимость от задачи:** Эта задача выполняется после того, как Вальковец наполнит ТЗ содержанием. Если содержание еще не готово, нужно дождаться его завершения.

**Фокус на форматировании:** Задача не касается содержательной части требований (это зона ответственности Вальковца). Основная цель — привести документ к единым стандартам оформления.

**Применение правил:** Используй созданные ранее правила как чеклист для проверки. Если правила что-то не покрывают, действуй по здравому смыслу и сохраняй единообразие.
