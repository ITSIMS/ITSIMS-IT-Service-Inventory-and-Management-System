# Провести формальную инспекцию системных требований

**Создана:** 2025-12-09
**Ответственный:** Зацепилин

## Описание

Необходимо провести формальную инспекцию документа системных требований, созданного Вальковцем. Инспекция должна выявить все дефекты документа и обеспечить его высокое качество перед утверждением.

**Контекст:**
- Вальковец создаст первую версию системных требований
- Для инспекции будет использован специальный шаблон
- Инспекция аналогична успешным инспекциям технического задания

**Зависимости:**
- Дождаться завершения задачи Вальковца (создание системных требований)
- Дождаться завершения создания шаблона инспекции (собственная задача)

**Ожидаемый результат:**
- Проведена полная формальная инспекция документа
- Создан отчет инспекции `inspections/system_requirements/inspection_v1_0.md`
- Все дефекты зафиксированы и классифицированы
- Даны рекомендации по исправлению
- Принято решение: утвердить / доработать

**Проверяемый документ:**
- `artifacts/2_system_requirements.md` версии 1.0

## Примечания

**Систематичность:** Используй шаблон инспекции полностью, проходя по всем чек-листам. Это обеспечит полноту проверки.

**Объективность:** Фиксируй все дефекты, даже незначительные. Классификация по критичности поможет приоритизировать исправления.

**Конструктивность:** Для каждого дефекта давай конкретную рекомендацию по исправлению. Это поможет Вальковцу быстро устранить проблемы.

**Опыт инспекций ТЗ:** Используй опыт успешных инспекций технического задания, применяя те же принципы систематичности и тщательности.

**Ключевые аспекты для проверки:**
1. **Качество требований** — однозначность, измеримость, проверяемость
2. **Трассировка** — полное покрытие требований из ТЗ
3. **Идентификация** — корректность и уникальность идентификаторов
4. **Полнота** — все аспекты системы описаны

**Связь с доработкой:** Если документ потребует доработки, Вальковец получит четкий список дефектов и рекомендаций для исправления. После исправления может потребоваться повторная инспекция.
