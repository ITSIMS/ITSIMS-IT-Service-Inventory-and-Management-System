## Описание

Необходимо провести формальную инспекцию таблиц трассировки требований, подготовленных Когановским. Инспекция должна проверить корректность связей между требованиями разных уровней и полноту покрытия.

**Контекст:**

- Инспекция проводится отдельно от инспекции документа системных требований

**Зависимости:**
- Дождаться завершения задачи Когановского (обновление таблиц трассировки)

- Проведена полная формальная инспекция таблиц трассировки
- Создан отчет инспекции `inspections/traceability/inspection_v1_0.md`

- Подсчитаны метрики покрытия
- Даны рекомендации по исправлению
- Принято решение: утвердить / доработать

**Проверяемые документы:**
- `artifacts/traceability/TS_to_SR.md`

## Примечания

**Критичность трассировки:** Таблицы трассировки — ключевой инструмент управления требованиями. Их качество напрямую влияет на:
- Полноту реализации требований
- Возможность анализа влияния изменений
- Валидацию и верификацию системы
- Соответствие регуляторным требованиям

**Цель инспекции:** Обеспечить 100% покрытие требований на каждом уровне и корректность всех связей.

**Типичные дефекты:**
- **Отсутствие связи** — требование не покрыто
- **Некорректная связь** — требование связано с неправильным элементом
- **Несуществующий ID** — ссылка на несуществующее требование
- **"Висящее" требование** — требование без источника

**Метрики покрытия:**
- **ТЗ → СТ:** Процент требований из ТЗ, имеющих связи с системными требованиями
- **СТ → ТЗ:** Процент системных требований, связанных с требованиями из ТЗ
- **Цель:** 100% в обоих направлениях

**Систематичность:** Используй шаблон полностью, проверяя каждый аспект. Это особенно важно для таблиц трассировки, где легко пропустить дефекты.

**Связь с доработкой:** Если таблицы потребуют доработки, Когановский получит четкий список дефектов. После исправления может потребоваться повторная инспекция.
