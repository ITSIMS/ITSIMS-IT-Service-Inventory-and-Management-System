## Описание

Проанализировать все функциональные модули системы ITSIMS и определить два наиболее подходящих для полной верификации (формальная инспекция, тестирование и пр.).

**Критерии выбора:**
- Модуль не должен быть банальным (не просто CRUD или UI)
- Модуль должен быть уникальным для нашего проекта
- Модуль должен покрываться разными видами верификации
- Модуль должен иметь чёткие границы
