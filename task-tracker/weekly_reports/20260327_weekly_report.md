1. **Подготовка к демонстрации:** определены докладчики по обоим кандидатам (dependency_graph и rbac) и ответственный за фиксацию замечаний; все назначения зафиксированы в чеклисте готовности.

3. **Целостность документации:** проверены и исправлены перекрёстные ссылки между всеми демонстрационными документами; оформлен порядок показа с таймингом и переходами между артефактами.

4. **Готовность к вопросам:** подготовлены развёрнутые ответы на ожидаемые вопросы по покрытию требований, рискам и ограничениям для обоих кандидатов.

5. **Dry-run и план КУ:** проведён dry-run демонстрации по сценариям RBAC, зафиксированы замечания; схема эскалации по RBAC-инцидентам в плане КУ актуализирована.

6. **Демо-модуль «Каталог IT-сервисов»:** разработан полноценный рабочий прототип системы в папке `demo/`. Стек: Go (Gin) + PostgreSQL на бэкенде, React + TypeScript + Vite на фронтенде, Docker Compose для развёртывания одной командой. Реализован полный CRUD: `GET/POST /api/v1/services`, `GET/PUT/DELETE /api/v1/services/:id`. Написано **48 модульных тестов** (handler: 100%, service: 100%, repository: 95.9%, итого **98.5%** покрытия кода).
