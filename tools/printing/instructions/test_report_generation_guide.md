# Руководство по генерации отчётных артефактов по результатам тестов

| Дата       | Номер версии | Автор   | Комментарий                                     |
| ---------- | ------------ | ------- | ----------------------------------------------- |
| 2026-03-04 | v1.0         | Ловков  | Описание генерации протоколов и отчётов         |

---

## 1. Цель документа

Документ описывает, как на основе сценариев и результатов прогона тестов **полуавтоматически формировать**:

- отчёты о прогонах;
- протоколы проверки;
- отчёты о дефектах.

---

## 2. Входные артефакты

- Матрица покрытия: `validations/6_verification_candidates/2026-02-18_verification_coverage_matrix.md`
- Сценарии кандидатов: `validations/6_verification_candidates/2026-02-19_verification_candidates.md`
- Отчёты о прогонах по шаблону: `validations/6_verification_candidates/2026-03-02_test_run_report_template.md`

---

## 3. Процесс генерации

1. **Подготовка данных**  
   - обновить сценарии и матрицу покрытия;  
   - выполнить прогон и заполнить отчёт по шаблону.

2. **Формирование протокола и баг-репортов**  
   - на основе заполненного отчёта о прогонах сформировать протокол и список дефектов по шаблону `2026-03-03_test_protocol_and_bug_report_template.md`;  
   - сгруппировать дефекты по типам и приоритетам.

3. **Экспорт в PDF (по необходимости)**  
   - использовать скрипты из `tools/printing/prepare_for_print.sh` или `prepare_all_for_print.sh` для конвертации отчётов в PDF.

---

## 4. Результирующие артефакты

На выходе процесса генерации получаются:

- обновлённые Markdown-отчёты (прогоны, протоколы, баг-репорты);
- при необходимости — PDF-версии для сдачи.
