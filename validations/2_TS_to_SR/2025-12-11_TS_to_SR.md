| 2   | **Формат идентификаторов**           | Все идентификаторы требований в таблицах соответствуют установленным форматам: `TS-*` для ТЗ, `SR-*` для Системных требований.                                                       |              |           |

### Часть II: Полнота трассировки вверх (SR → TS)

| №   | Критерий проверки                             | Описание                                                                                                                                      | Статус (+/-) | Замечания |
| --- | --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- | :----------: | --------- |
| 5   | **Полнота покрытия SR в таблице трассировки** | Все требования из документа "Системные требования" (`SR-*`) присутствуют в "Таблице 2: Трассировка вверх".                                    |              |           |
| 6   | **Связь каждого SR с ТЗ**                     | Для каждого требования SR указана связь хотя бы с одним требованием из ТЗ (`TS-*`). Отсутствуют "висящие" требования SR без трассировки к ТЗ. |              |           |
| 7   | **Обоснованность связей SR → TS**             | Связи между требованиями SR и ТЗ логичны: требование SR действительно детализирует указанное требование ТЗ на системном уровне.               |              |           |

### Часть III: Полнота трассировки вниз (TS → SR)

| №   | Критерий проверки                             | Описание                                                                                                                                                                               | Статус (+/-) | Замечания |
| --- | --------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------: | --------- |
| 8   | **Полнота покрытия ТЗ в таблице трассировки** | Все требования из документа "Техническое задание" (`TS-*`) присутствуют в "Таблице 1: Трассировка вниз".                                                                               |              |           |
| 9   | **Раскрытие каждого TS в SR**                 | Для каждого требования ТЗ указана связь хотя бы с одним требованием SR (или символ "—" с пояснением, почему требование ещё не раскрыто). Нет требований ТЗ без плана детализации в SR. |              |           |
| 10  | **Достаточность детализации TS → SR**         | Требования ТЗ раскрыты на системном уровне в SR с достаточной детализацией. Высокоуровневые требования конкретизированы с указанием атрибутов, действий, условий.                      |              |           |
| 11  | **Покрытие всех типов требований из ТЗ**      | Все типы требований из ТЗ (функциональные, нефункциональные, интеграции) раскрыты соответствующими требованиями в SR.                                                                  |              |           |

### Часть IV: Согласованность и непротиворечивость

| №   | Критерий проверки                         | Описание                                                                                                                                  | Статус (+/-) | Замечания |
| --- | ----------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- | :----------: | --------- |
| 12a | **Прямая симметричность трассировки**     | Все связи из Таблицы 1 (прямая трассировка) присутствуют в Таблице 2 (обратная трассировка).                                              |              |           |
| 12b | **Обратная симметричность трассировки**   | Все связи из Таблицы 2 (обратная трассировка) присутствуют в Таблице 1 (прямая трассировка).                                              |              |           |
| 13  | **Отсутствие дублирования связей**        | В таблицах нет дублирующихся связей между одними и теми же требованиями.                                                                  |              |           |
| 14  | **Корректность множественных связей**     | Если одно требование ТЗ детализировано несколькими требованиями SR (или наоборот), все связи указаны корректно и логичны. |              |           |
| 15  | **Отсутствие противоречий в требованиях** | Требования SR, связанные с одним требованием ТЗ, не противоречат друг другу и не дублируют функциональность.              |              |           |

### Часть V: Качество покрытия требований

| №   | Критерий проверки                                 | Описание                                                                                                                                                                          | Статус (+/-) | Замечания |
| --- | ------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------: | --------- |
| 16  | **Полнота раскрытия функциональных требований**   | Все функциональные требования из ТЗ (`TS-F*`) детализированы в SR с достаточной полнотой. Не упущены важные аспекты и детали.                                                     |              |           |
| 17  | **Полнота раскрытия нефункциональных требований** | Все нефункциональные требования из ТЗ (`TS-NF*`) детализированы в SR. Требования к производительности, надёжности, безопасности, масштабируемости раскрыты на системном уровне.   |              |           |
| 18  | **Отсутствие избыточных требований в SR**         | Все требования SR трассируются к ТЗ. Нет требований, выходящих за рамки проекта, определённые в ТЗ.                                                                               |              |           |
| 19  | **Соответствие уровня детализации**               | Требования SR находятся на правильном уровне детализации (системный уровень): они детализируют ТЗ, но не углубляются в технические детали реализации (которые должны быть в SWR). |              |           |

## Таблица обнаруженных проблем

| ID     | Тип проблемы | Требование(я) | Описание проблемы | Рекомендация по устранению | Статус |
| ------ | ------------ | ------------- | ----------------- | -------------------------- | ------ |
| TR-001 |              |               |                   |                            |        |
| TR-002 |              |               |                   |                            |        |
| TR-003 |              |               |                   |                            |        |

**Типы проблем:**
- **Отсутствие связи** — требование не имеет трассировки (висящее требование)
- **Неполнота детализации** — требование ТЗ раскрыто недостаточно подробно в SR
- **Несогласованность таблиц** — прямая и обратная таблицы трассировки противоречат друг другу
- **Избыточность** — требование SR выходит за рамки требований ТЗ
- **Некорректная связь** — связь между требованиями логически неверна
- **Несуществующий ID** — ссылка на несуществующее требование в артефакте
- **Дублирование связей** — одна и та же связь указана несколько раз в таблице
- **Ошибка в якоре** — якорь не соответствует ID требования или отсутствует
- **Битая ссылка** — ссылка ведет не на тот якорь или якорь не существует

**Статусы:**
- **Открыта** — проблема обнаружена, требует устранения
- **Устранена** — проблема исправлена автором
- **Отклонена** — проблема признана несущественной
