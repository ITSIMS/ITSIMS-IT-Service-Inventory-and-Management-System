## 3. Шаблон баг-репорта

Допустимые статусы дефекта: `Открыт` | `В работе` | `Закрыт` | `Отложен`.

### 3.1 Поля баг-репорта

- **ID дефекта:** `DEF-XXX`
- **Заголовок:** краткое описание проблемы
- **Тип:** Функциональный / UI / Производительность / Безопасность / Документация / Прочее
- **Приоритет:** High / Medium / Low
- **Статус:** из закрытого списка выше
- **Окружение:** стенд, браузер, версия
- **Предусловия**
- **Шаги воспроизведения**
- **Фактический результат**
- **Ожидаемый результат**
- **Связанные требования:** TS-*, SR-*, SWR-*
- **Связанные тесты:** ID тест-кейсов

### 3.2 Табличный формат регистрации дефектов

| ID     | Тип            | Приоритет (H/M/L) | Краткое описание | Связанные требования | Связанные тесты | Статус  |
| ------ | -------------- | ----------------- | ---------------- | -------------------- | --------------- | ------- |
| DEF-01 | Функциональный | H                 | ...              | TS-..., SR-..., ...  | TC-...          | Открыт  |

---

## 4. Итоговое заключение

Итоговое заключение по результатам проверки формулируется в одном из вариантов:

- **Принять без замечаний** — блокирующих дефектов нет, критерии качества выполнены;
- **Принять с доработками** — есть дефекты ≤ Medium, сформирован план доработок;
- **Отклонить** — есть дефекты High без ясного плана исправления или существенные пробелы покрытия.
