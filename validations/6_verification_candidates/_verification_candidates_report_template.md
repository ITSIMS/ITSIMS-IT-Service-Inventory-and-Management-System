## 1. Цель проверки

Кратко сформулируйте цель верификации кандидата:

- подтверждение соответствия требованиям;
- проверка полноты сценариев и тестового покрытия;
- фиксация дефектов и рисков перед демонстрацией.

**Обязательно выбрать ровно один вариант** (допустимы только эти три):

| Вариант | Условие выбора |
| ------- | -------------- |
| [ ] **Принять без замечаний** | Все критерии — `+`, дефектов нет, рисков нет |
| [ ] **Принять с доработками** | Есть дефекты ≤ Medium и/или критерии `+/-`; план доработок заполнен |
| [ ] **Отклонить** | Есть дефекты High без митигирующих действий или критерии `-` |
