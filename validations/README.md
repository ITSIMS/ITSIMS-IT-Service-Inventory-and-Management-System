## 1. Цель документа

Данный документ описывает организацию директории `validations/`, содержащей отчеты проверок и формальных инспекций документов проекта.



## 2. Структура директории

Для каждого артефакта создается отдельная папка. Внутри каждой папки:
- Шаблон для проведения инспекции (файл с префиксом `_`)
- Отчеты инспекций в формате: `ГГГГ-ММ-ДД_<артефакт>.<расширение>`

```
validations/
├── README.md
├── 1_technical_specification/        # Техническое задание
│   ├── _inspection_template.tex
│   ├── _inspection_template.pdf
│   ├── 2025-11-03_technical_specification.tex
│   └── 2025-11-03_technical_specification.pdf
└── 2_TS_to_SR/                       # Таблица трассировки требований
    ├── _inspection_template.tex
    ├── _inspection_template.pdf
    ├── 2025-12-10_TS_to_SR.tex
    └── 2025-12-10_TS_to_SR.pdf
```

**Правила именования отчетов:**
- Формат: `<YYYY-MM-DD>_<артефакт>.<расширение>`
- `<YYYY-MM-DD>` — дата проведения валидации
- `<артефакт>` — название артефакта (совпадает с именем артефакта в `artifacts/`)
- `<расширение>` — `.pdf`, `.tex`, или `.md`



## 3. Форматы отчетов

Отчеты могут быть в одном из следующих форматов:

| Формат  | Когда использовать                          |
| ------- | ------------------------------------------- |
| **PDF** | Финальная версия для сдачи и архива         |
| **TeX** | Исходник LaTeX документа для редактирования |
| **MD**  | Отчеты в формате Markdown для простоты      |

**Рекомендация:** Хранить и PDF, и исходник (TeX или MD) для возможности правок.
