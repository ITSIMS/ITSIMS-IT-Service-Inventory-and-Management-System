## 1. Цель документа

Данный документ описывает организацию директории `validations/`, содержащей отчеты проверок и формальных инспекций документов проекта.



## 2. Структура директории

Для каждого артефакта создается отдельная папка. Внутри каждой папки:
- Шаблон для проведения проверки (файл с префиксом `_`)
- Отчеты проверок в формате: `ГГГГ-ММ-ДД_<артефакт>.md`

**Типы шаблонов:**
- `_inspection_template.md` — для формальной инспекции артефактов (ТЗ, Системные требования)
- `_traceability_analysis_template.md` — для анализа трассировки между требованиями разных уровней
- `_verification_candidates_report_template.md` — для унифицированного отчета по проверке кандидатов верификации

```
validations/
├── README.md
├── 1_technical_specification/        # Техническое задание
│   ├── _inspection_template.md
│   └── 2025-11-03_technical_specification.md
├── 2_TS_to_SR/                       # Анализ трассировки ТЗ ↔ SR
│   ├── _traceability_analysis_template.md
│   └── 2025-12-10_TS_to_SR.md
├── 3_system_requirements/            # Системные требования
│   ├── _inspection_template.md
│   └── 2025-12-XX_system_requirements.md
└── 6_verification_candidates/        # Демонстрационные сценарии верификации кандидатов
    ├── 2026-02-18_verification_coverage_matrix.md
    ├── _verification_candidates_report_template.md
    └── 2026-02-19_verification_candidates.md
```

**Правила именования отчетов:**
- Формат: `<YYYY-MM-DD>_<артефакт>.md`
- `<YYYY-MM-DD>` — дата проведения валидации
- `<артефакт>` — название артефакта (совпадает с именем артефакта в `artifacts/`)
- `<расширение>` — `.md` (Markdown)



## 3. Формат отчетов

Все отчеты о формальной инспекции создаются в формате **Markdown (`.md`)**.

**Почему Markdown:**
- ✅ Легко редактировать и вносить правки
- ✅ Понятные изменения в Git diff
- ✅ Удобно просматривать на GitHub
- ✅ Простота работы с таблицами критериев проверки
- ✅ При необходимости легко конвертировать в PDF через Pandoc

**Конвертация в PDF:** Для сдачи документов в формате PDF используйте Pandoc (см. [инструкцию](../tools/printing/instructions/markdown_to_pdf_guide.md)).
