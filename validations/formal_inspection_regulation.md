## Введение

Формальная инспекция — это процесс систематической проверки артефактов проекта (документации, кода, спецификаций) с целью выявления дефектов и несоответствий установленным стандартам.

Данный регламент определяет порядок проведения формальной инспекции в проекте ITSIMS.

---

## Цель формальной инспекции

Основные цели:

1. **Раннее обнаружение дефектов** — выявление ошибок на этапе разработки документации
2. **Повышение качества артефактов** — обеспечение соответствия стандартам
3. **Обмен знаниями** — распространение лучших практик в команде
4. **Снижение затрат** — предотвращение дорогостоящих исправлений на поздних этапах

---

## Участники процесса

| Роль          | Обязанности                                      |
| ------------- | ------------------------------------------------ |
| **Модератор** | Организует и ведёт процесс инспекции             |
| **Автор**     | Создатель инспектируемого артефакта              |
| **Инспектор** | Проверяет артефакт и фиксирует дефекты           |
| **Читатель**  | Зачитывает артефакт по частям (опционально)      |
| **Протоколист** | Документирует найденные дефекты и решения      |

**Требования:**
- Минимум 3 участника на инспекцию
- Автор не может быть модератором
- Все участники должны быть подготовлены

---

## Этапы проведения формальной инспекции

### 1. Планирование

**Ответственный:** Модератор

**Действия:**
- Определить объект инспекции
- Выбрать участников
- Назначить дату и время
- Подготовить чек-листы и критерии проверки

**Результат:** Уведомление участников с указанием артефакта для проверки

### 2. Подготовка

**Ответственный:** Все участники

**Действия:**
- Изучить инспектируемый артефакт
- Ознакомиться с чек-листом и стандартами
- Подготовить список вопросов и замечаний

**Результат:** Готовность участников к проведению инспекции

**Временные затраты:** 1-2 часа индивидуальной работы

### 3. Проведение инспекции (собрание)

**Ответственный:** Модератор

**Действия:**
1. Открыть собрание и представить цели
2. Пошаговый обзор артефакта
3. Фиксация каждого найденного дефекта
4. Классификация дефектов по критичности
5. Определение решений по каждому дефекту

**Результат:** Протокол инспекции с перечнем дефектов

**Длительность:** 1-2 часа

**Классификация дефектов:**
- **Критический** — блокирует дальнейшую работу
- **Значительный** — требует обязательного исправления
- **Незначительный** — рекомендуется исправить
- **Замечание** — предложение по улучшению

### 4. Доработка

**Ответственный:** Автор

**Действия:**
- Исправить выявленные дефекты
- Внести правки согласно решениям
- Подготовить обновлённую версию артефакта

**Результат:** Исправленный артефакт

### 5. Верификация

**Ответственный:** Модератор или назначенный инспектор

**Действия:**
- Проверить, что все дефекты устранены
- Убедиться в корректности исправлений

**Результат:** Подтверждение устранения дефектов

### 6. Завершение

**Ответственный:** Модератор

**Действия:**
- Оформить итоговый отчёт
- Архивировать протокол и отчёт
- Уведомить команду о результатах

**Результат:** Отчёт о формальной инспекции (см. шаблон в `_formatting/formal_inspection_report_template.pdf`)

## Оформление результатов

### Протокол инспекции

Документ, фиксирующий ход проведения инспекции:

- Дата и время проведения
- Список участников и их роли
- Инспектируемый артефакт (название, версия)
- Перечень найденных дефектов с классификацией
- Решения по каждому дефекту
- Подписи участников

### Отчёт о формальной инспекции

Итоговый документ по результатам инспекции (см. `_formatting/formal_inspection_report_template.pdf`):

- Резюме инспекции
- Статистика по дефектам
- Статус устранения дефектов
- Рекомендации для дальнейшей работы

**Место хранения:** `validations/reports/`

**Формат имени:** `ГГГГ-ММ-ДД_<артефакт>_formal_inspection_report_vX.Y.pdf`

**Пример:** `2025-11-03_spec_technical_formal_inspection_report_v1.0.pdf`

---

## Критерии завершения

Инспекция считается завершённой, если:

1. ✅ Все найденные **критические** и **значительные** дефекты устранены
2. ✅ Проведена верификация исправлений
3. ✅ Оформлен итоговый отчёт
4. ✅ Отчёт размещён в репозитории (`validations/reports/`)
5. ✅ Артефакт обновлён с указанием новой версии

**Решение об успешном завершении принимает модератор инспекции.**
